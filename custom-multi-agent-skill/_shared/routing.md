# Routing Decision Tree (하이브리드 모드 분기 가이드)

본 문서는 오케스트레이터가 사용자 인풋의 성격을 분석하여, **[일반 대화/기획 모드]**, **[일반 API 코딩 모드]**, **[리팩토링 모드]**로 자동 분기하기 위한 라우팅 규칙입니다.

---

## 1. 태스크 분류 및 라우팅 흐름

```
                          [사용자 요청 수신]
                                  │
         ┌────────────────────────┼────────────────────────┐
         ▼ (단순 질의/가벼운 대화)     ▼ (구현/코딩 지시)        ▼ (대규모 아키텍처 변경)
   [일반 대화/기획 모드]         [일반 API 코딩 모드]         [리팩토링 모드]
   - 비용: $0 (무료 쿼터)        - 비용: 소량 API 과금        - 비용: 중대형 API 과금
   - 모델: Gemini Flash         - 모델: Fable 5 워커        - 모델: Fable 5 + GPT-5.5
   - 도구: 내장 대화            - 도구: call_worker.py      - 규칙: 카파시 3원칙 일시 완화
```

---

## 2. 세부 라우팅 규칙 (Routing Rules)

### 2.1 일반 대화 / 기획 모드 (Conversation Mode)
* **진입 조건**:
  * 단순 질문 ("이 함수가 무슨 뜻이야?", "Flask 사용법 알려줘")
  * 개념 설계 및 브레인스토밍 단계
  * 태스크 기획서 작성 및 `task.md` 초기화 단계
* **동작 지침**:
  * 외부 API(`call_worker.py`)를 절대 호출하지 마십시오.
  * 오케스트레이터 자체(Gemini 3.5 Flash)의 내장 추론 및 무료 쿼터를 활용하여 유저에게 즉시 답변하십시오.

### 2.2 일반 API 코딩 모드 (Standard Coding Mode)
* **진입 조건**:
  * 단일 파일 내 기능 추가 및 버그 수정
  * 유닛 테스트 코드 작성
  * 간단한 스크립트 작성
* **동작 지침**:
  * 카파시 3원칙(외과수술식 수정, 인접 코드 보존)을 엄격히 적용합니다.
  * `call_worker.py`를 사용하여 **`claude-fable-5`** 워커를 1회 가동하고 코드를 수령하십시오.
  * 결과 검증을 위해 `validate_result.py`로 글자 수 제한(brief 1200자)을 린트하십시오.

### 2.3 리팩토링 모드 (Refactoring Mode)
* **진입 조건**:
  * 다중 파일에 걸친 클래스/모듈 재설계
  * 파일 구조 및 디렉토리 이주 (Migration)
  * 레거시 코드 전체에 대한 광범위한 청소 및 최적화
* **동작 지침**:
  * 카파시 3원칙을 일시적으로 완화하며 구조적 리팩토링을 허용합니다.
  * **1단계**: 코드를 수정하기 전, `design-basis.md`에 리팩토링 설계서를 정의하고 사용자에게 명시적으로 승인받으십시오.
  * **2단계**: `call_worker.py`로 **`claude-fable-5`** 워커를 호출하여 리팩토링된 코드를 수령하고, **`gpt-5.5`** 크리틱 워커를 연속 가동하여 교차 비평(Cross-Review)을 강제하십시오.
  * **3단계**: `validate_result.py` 검증 후 기존 레거시 파일을 아카이빙(백업)하고 신규 구조화된 파일로 안전하게 대체하십시오.
